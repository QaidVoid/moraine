# Test corpus

Moraine is benchmarked and validated against a snapshot of real Gentoo data so
its results and timings can be compared directly with stock Portage. The corpus
lives in the git-ignored `corpus/` directory at the repository root and is never
committed; each contributor populates it from a real Gentoo system.

## Convention

`MORAINE_CORPUS` points at a captured **system root** (an `EROOT`), laid out
exactly like a real `/`. Every corpus-gated test and benchmark reads its input
from a standard sub-path of that root and **skips cleanly when the sub-path is
absent**, so a partial corpus (for example vdb-only) still leaves the suite
green. The variable being unset skips everything.

On a live Gentoo system the root is simply `/`:

```sh
MORAINE_CORPUS=/ cargo test --workspace
```

### Standard sub-paths

| Sub-path | Used by | Notes |
|----------|---------|-------|
| `etc/portage/repos.conf` | repo discovery, resolution, `@world` | a file or a `repos.conf` directory |
| `etc/portage/make.conf` | config parsing | |
| `etc/portage/make.profile` | `@world`/pretend tests | must resolve (a broken symlink skips those tests) |
| `var/db/pkg` | installed-store import and round-trip | the vdb |
| `var/cache/binpkgs` | binpkg round-trip | searched recursively; absent or empty skips |
| `var/cache/moraine/repos` | — | generated by the harness, not part of the corpus |

## Populating from a running Gentoo system

Mirror the system paths under `corpus/` so the layout matches a real root. The
key subtlety is `make.profile`: rsync copies it as a **symlink**, and the
system's symlink usually points at `../../var/db/repos/<repo>/profiles/...`
(relative). For that to resolve inside the corpus, the repository must live at
`corpus/var/db/repos/<repo>` (not `corpus/repos/<repo>`).

```sh
mkdir -p corpus/etc corpus/var/db corpus/var/db/repos

# Configuration (make.conf, repos.conf, make.profile, package.* files).
rsync -a --delete /etc/portage/ corpus/etc/portage/

# Installed package database (vdb).
rsync -a --delete /var/db/pkg/ corpus/var/db/pkg/

# The repository trees referenced by repos.conf, at the path make.profile
# expects. The md5-cache under each repo's metadata/ is what the resolver reads.
rsync -a --delete /var/db/repos/gentoo/ corpus/var/db/repos/gentoo/

# Optional: binary packages, for the binpkg round-trip test.
rsync -a --delete /var/cache/binpkgs/ corpus/var/cache/binpkgs/
```

A config-and-vdb corpus (just `etc/portage` and `var/db/pkg`) is enough to
exercise the installed-store and config paths; the repo, resolution, and
`@world` tests skip until the repository trees are present.

### Avoiding the broken make.profile

If `corpus/etc/portage/make.profile` does not resolve, the `@world`/pretend
tests skip rather than fail. Two ways to make it resolve:

- Copy the repository to `corpus/var/db/repos/<repo>` as above, so the relative
  symlink resolves inside the corpus (self-contained).
- Or point it at the live tree:
  `ln -snf "$(readlink -f /etc/portage/make.profile)" corpus/etc/portage/make.profile`
  (resolves against the running system; not self-contained).

### repos.conf and hermeticity

A captured `repos.conf` usually has absolute `location = /var/db/repos/<repo>`
lines, so discovery resolves against the **live** system trees rather than the
copies under `corpus/`. That is fine on the machine the corpus came from. For a
fully self-contained corpus, rewrite the `location` lines to the
`corpus/var/db/repos/<repo>` copies.

## Comparing against stock Portage

Set `MORAINE_CORPUS_PORTAGE=1` (with `portageq`/`emerge` on `PATH`) to diff
Moraine's resolved repository order and selection against Portage. Set
`MORAINE_CORPUS_ATOM=<atom>` to choose the package the resolution harness
resolves (default `sys-apps/portage`).

```sh
MORAINE_CORPUS=/ MORAINE_CORPUS_PORTAGE=1 cargo test --workspace
```

## Driving the binary directly

For interactive checks rather than assertions, run the `moraine` binary against
the corpus root:

```sh
cargo run -p moraine-cli -- --root "$MORAINE_CORPUS" --config-root "$MORAINE_CORPUS" -pv sys-apps/portage
```
